******* Метод listFiles класса File *******

1. public File[] listFiles() - Возвращает массив абстрактных путей, обозначающих файлы
                               в каталоге, обозначенном этим абстрактным путем.

Если этот абстрактный путь не обозначает каталог, то этот метод возвращает null. В противном
случае возвращается массив объектов File, по одному для каждого файла или каталога в каталоге.
Пути, обозначающие сам каталог и родительский каталог каталога, не включаются в результат.
Каждый результирующий абстрактный путь создается из этого абстрактного пути с помощью
конструктора File(File, String) (см. FileСonstructor.txt). Следовательно, если этот путь
абсолютный, то каждый результирующий путь абсолютен; если это имя пути является относительным,
то каждое результирующее имя пути будет относиться к одному и тому же каталогу.

Нет никакой гарантии, что строки имен в результирующем массиве будут отображаться в каком-либо
определенном порядке; в частности, не гарантируется их появление в алфавитном порядке.

!!! Внимание !!! Класс Files определяет метод newDirectoryStream для открытия каталога
и перебора имен файлов в каталоге.
------------------------------------------------------------------------------------------------
См. https://github.com/JcoderPaul/JavaExtended-30-32/blob/master/Less_31_ch_2_Files/src/Less_31_ch_2_Files/ReadMe.txt
------------------------------------------------------------------------------------------------
Это позволяет использовать меньше ресурсов при работе с очень большими каталогами.

Возвращает: Массив абстрактных путей, обозначающих файлы и каталоги в каталоге, обозначенном
            этим абстрактным путем. Массив будет пустым, если каталог пуст. Возвращает null,
            если это абстрактное имя пути не обозначает каталог или если возникает ошибка
            ввода-вывода.
Исключения: SecurityException — если диспетчер безопасности существует и его метод
            SecurityManager.checkRead(String) запрещает доступ для чтения к каталогу.
------------------------------------------------------------------------------------------------

2. public File[] listFiles(FilenameFilter filter)

Возвращает массив абстрактных имен путей, обозначающих файлы и каталоги в каталоге, обозначенном этим
абстрактным путем, которые удовлетворяют указанному фильтру. Поведение этого метода такое же, как у
метода listFiles(), за исключением того, что пути в возвращаемом массиве должны удовлетворять фильтру.
Если данный фильтр равен нулю, то принимаются все пути. В противном случае путь удовлетворяет фильтру
тогда и только тогда, когда значение true получается, когда метод фильтра - FilenameFilter.accept(File, String)
вызывается для этого абстрактного пути и имени файла или каталога в каталоге, который он обозначает.

Параметры: filter - фильтр имени файла;
Возвращает: Массив абстрактных путей, обозначающих файлы и каталоги в каталоге, обозначенном этим
            абстрактным путем. Массив будет пустым, если каталог пуст. Возвращает null, если это
            абстрактное имя пути не обозначает каталог или если возникает ошибка ввода-вывода.
Исключения: SecurityException — если диспетчер безопасности существует и его метод
            SecurityManager.checkRead(String) запрещает доступ для чтения к каталогу.
------------------------------------------------------------------------------------------------

3. public File[] listFiles(FileFilter filter)

Возвращает массив абстрактных имен путей, обозначающих файлы и каталоги в каталоге, обозначенном

фильтру тогда и только тогда, когда значение true получается при вызове метода фильтра
FileFilter.accept(File) для пути.

Параметры: filter - файловый фильтр;
Возвращает: Массив абстрактных путей, обозначающих файлы и каталоги в каталоге, обозначенном этим
            абстрактным путем. Массив будет пустым, если каталог пуст. Возвращает null, если это

Исключения: SecurityException — если диспетчер безопасности существует и его метод
            SecurityManager.checkRead(String) запрещает доступ для чтения к каталогу.
